Opdracht 1

In de algemene informatie bij de labo's kon je al lezen dat het implementeren van algoritmes doorgaans veel sneller (en sneller foutloos) kan als je duidelijk voor ogen hebt hoe de gebruikte gegevensstructuur eruit ziet - en hoe je de gegevens erin moet aanspreken. Om je op weg te helpen stoffen we eerst je C++-kennis wat af met twee herhalingsoefeningen.
  1. De eerste gaat in op de keuze van het type pointer dat je gebruikt voor bepaalde toepassingen.
  2. De tweede introduceert geleidelijk de unique pointers van C++11 in een gelinkte lijst. De startcode kan je hier afhalen.

Feedback verwerken (huiswerk 13-14 febr)

Hier vind je een oplossing van het eerste deel van de tweede herhalingsoefening. Dit is een van de zeldzame keren dat je een mogelijke oplossing krijgt. Bekijk dit kritisch; jouw versie kan verschillend maar toch even veel waard zijn! Werk hierop verder om deel 2 en deel 3 te schrijven -- zorg dat alle voorgestelde tests slagen bij het overgaan op unique pointers.

Tip: op voorgaande pagina vind je bij de 'Hulpmiddelen' drie links naar tutorials over specifieke C++11-topics. Warm aanbevolen!

Opdracht 2 (12/15 febr 2016 en 17 febr 2016)

Je krijgt van ons een testprogrammaatje, lijsttest11.cpp dat oppervlakkig test of alle move- en copy-operatoren in lijst11.h behoorlijk werken. Vul de code van dit laatste bestand aan zodat ze de tests overleeft.

Opdracht 3 (19/22 febr 2016)

Zoals je in de twee uitschrijfoperatoren (operator<< en schrijf()) van lijst11.h kan zien maken iteratoren bepaalde code veel eenvoudiger, vooral sinds C++11. Definieer de nodige elementen om de eenvoudige code te kunnen gebruiken. De nodige elementen zijn alvast voorwaardelijk gedeclareerd in lijst11.h.
